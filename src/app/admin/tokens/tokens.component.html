Proxy Configuration

By default all client api requests are made to the webserver and then forwarded to the rest server instance. By modifying the properties below things like whether or not SSL or OAuth headers are sent to the rest server when specified in the client configuration can be configured.

Client Configuration

These are parameters that the webapp uses to connect to the senzing rest api server. In most cases the defaults should be correct. Some installations may require the api server use authentication, access tokens, oauth etc. for security purposes. Docker instances usually use container-to-container communication and so the client requests are made to the webapp itself and fetched from the other container.
